# AI Champion 职责指南

> AI Champion 是团队中推动 AI 驱动开发落地的核心骨干

---

## Champion 选拔标准

- 对 AI 工具有热情和好奇心
- 愿意学习和分享
- 在团队中有一定影响力
- 不一定是技术最强的，但一定是最积极尝试的

## 核心职责

### 1. 日常推广（每周 1-2 小时）

- 收集和整理团队中好用的 Prompt 模板
- 发现同事在使用中的困难，提供帮助
- 维护团队 Prompt 知识库（共享文档/Wiki）

### 2. 案例分享（每月 1 次）

- 在每周五站会的 AI 分享环节中积极发言
- 每月整理一篇"本月 AI 提效案例集"
- 包含：成功案例、失败案例、技巧总结

### 3. 问题收集与反馈（持续）

- 收集团队对 AI 工具的痛点和建议
- 定期向管理层反馈使用情况
- 推动工具配置的优化（如 Skill 模板、API Key 管理）

### 4. 新人引导（按需）

- 新入职同事的 AI 工具入门培训（1 对 1，30 分钟）
- 帮助新人配置环境
- 提供前两周的 Pair Programming 支持

---

## 激励机制

| 激励 | 说明 |
|------|------|
| 季度表彰 | 每季度评选"最佳 AI Champion" |
| 培训优先 | 优先参加外部 AI 技术培训/会议 |
| 技术影响力 | 在技术月报/年报中署名 |
| 工具预算 | 优先获得 AI 工具升级/新工具试用权 |

## 第一个月行动计划

### 第 1 周
- [ ] 确认团队所有人环境可用
- [ ] 建立团队 Prompt 知识库（飞书文档/Notion）
- [ ] 收集培训后的第一波反馈

### 第 2 周
- [ ] 组织第一次复训（半天，重点解决实际使用中的问题）
- [ ] 整理前两周的常见问题 FAQ
- [ ] 推动至少 3 个 MR 使用 OpenSpec

### 第 3-4 周
- [ ] 整理第一篇"AI 提效案例集"
- [ ] 向管理层汇报第一个月的采用情况
- [ ] 识别团队中的"高阻力个体"，1 对 1 沟通
